### Problem (나빴던 것)
- Mermaid 렌더링을 너무 오래 환경 설치 방향으로 밀었다. `mermaido`의 Chromium 의존성을 확인한 시점에 바로 정적 SVG 또는 CI 전용 렌더링으로 전환했으면 시간이 줄었을 것이다.
- signed URL을 수동으로 복사하면서 한 글자가 손상되어 다운로드 실패가 발생했다. 긴 URL을 사람이 옮기는 방식은 오류 가능성이 높다.
- 정규식 검색에서 따옴표와 한글 escape가 섞여 `rg` parse error가 반복됐다. 단순 문자열 검색이나 JSON 파싱을 먼저 쓰는 편이 맞았다.
- 실행 컨테이너가 `requirements.txt` 최신 상태와 달랐다. 패키지 설치 상태를 먼저 확인했지만, 그 뒤 임시 설치와 재실행이 여러 번 이어져 환경 재현성이 약해졌다.
- 회고 로그 추출 중 전체 로그 원문이 크게 출력되어 노이즈가 많았다. JSONL은 처음부터 구조화 파서로 필요한 필드만 뽑아야 한다.

### Try (개선해야 할 것)
- Mermaid 자산 처리 정책을 둘로 나누자. 간단한 구조도는 직접 SVG를 저장하고, Mermaid CLI 렌더링은 전용 이미지나 CI job에서만 수행하도록 명시한다.
- Kaggle MCP가 signed URL을 반환할 때는 URL을 직접 재입력하지 말고, 가능한 한 도구 출력에서 바로 저장하는 작은 스크립트나 파일 기반 전달을 사용한다.
- 노트북 출력 보존 검증 스크립트를 별도 유틸로 만들자. setup 셀, `# [정답]` 셀, `problem.ipynb` 출력 제거 여부를 반복적으로 검사할 수 있어야 한다.
- 컨테이너 검증 전 `requirements.txt`와 실행 컨테이너의 import 가능 여부를 한 번에 점검하는 preflight 셀/명령을 마련한다.
- 회고 스킬은 JSONL 파싱 스크립트를 갖추는 편이 낫다. 현재처럼 `Select-String`으로 큰 로그를 훑으면 시스템/개발자 메시지가 섞여 분석 품질이 흔들린다.
